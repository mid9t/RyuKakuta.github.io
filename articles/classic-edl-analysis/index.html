EDL Phase 2 — PDF Analysis & Evaluation

What the PDF is actually saying, mapped to actionable phase work

Framing

The PDF makes one overarching argument that supersedes all its specific recommendations:

Do not gate on a single uncertainty scalar. EDL's output is a higher-order object (α). Extract a panel of features from it and learn to predict expected benefit from more compute — EURcompute — rather than thresholding any single metric.

Every specific recommendation in the PDF is a consequence of this. Evaluating them against Phase 2 requires first establishing which issues are Phase 2 problems (final-layer relevance classification) and which are Phase 3 problems (early exit gating). The PDF explicitly maps this in its phase structure on pages 12–13, which aligns almost exactly with the phase structure of this project.

Issue 1 ERU/MI = 0.084

PDF Diagnosis

Pages 8–9 identify this failure mode precisely. The predictive entropy minus expected entropy decomposition (the ERU/MI calculation) is flagged with an explicit warning:

"Treat Uepi and Ualea as features, not ground truth semantics."

The PDF further explains why ERU/MI is near zero in single-model EDL: without ensemble disagreement or density awareness, the Dirichlet distribution concentrates deterministically. The mutual information term collapses because there is no source of parameter uncertainty in the model — every forward pass produces identical α values.

PDF's Proposed Fix

Density-aware scaling (DAEDL-style, page 5): Multiply logits by a feature-space density estimate s(x) so that inputs far from the training manifold drive toward uniform predictions. This directly raises ERU/MI on OOD and hard negatives without requiring an ensemble.

Teacher uncertainty (page 6): Run a small deep ensemble offline on a sample of traffic, then train the EDL head to match not just labels but also the teacher's uncertainty ordering. This is distillation of epistemic signal — cheaper than full Bootstrap Distill but achieves similar calibration.

Evaluation for Phase 2

DAEDL-style density scaling is feasible in Phase 2 but requires computing a feature-space density estimate during training. The simplest implementation uses the CLS embeddings from the frozen BERT layers — a Gaussian discriminant analysis over the training set's CLS representations gives s(x) cheaply. The cost is one additional pass over the training set to fit the density model.

The teacher uncertainty approach is strictly a Phase 3 concern — it requires offline ensemble inference on a traffic sample, which presupposes a deployed system.

DAEDL density scaling is Phase 2 feasible and directly addresses ERU/MI. Teacher uncertainty distillation is Phase 3 only.

Issue 2 Routing Margin = 1.28 S Units

PDF Diagnosis

The routing margin problem is reframed entirely. Pages 9–10 argue that thresholding on S directly is the wrong approach regardless of margin width:

"Don't gate on 'epistemic < τ' directly. EDL can be overconfident under shift unless you explicitly fix it."

Instead, the PDF proposes EURcompute — learning to predict the expected benefit of spending more compute (deeper layers) from a panel of features. The margin problem then becomes: is the feature panel discriminative enough that a learned gating model can separate the slices?

From this perspective, the current routing margin of 1.28 is not necessarily fatal — it might be sufficient as one feature among many in a learned gating model, even if it is too narrow to threshold directly.

PDF's Proposed Fix

C-EDL conflict calibration: Measure representational disagreement using label-preserving transforms (paraphrases, sentence order perturbations). Passages near the semantic decision boundary (hard negatives) show higher disagreement across transforms than clear positives or easy negatives. This adds a conflict/dissonance feature orthogonal to S.

Margin metric (page 8): m(ℓ) = p̂₁(ℓ) − p̂₂(ℓ) — the gap between the top two predicted class probabilities. This is cheap to compute and provides a complementary signal to S. A hard negative with S=5.6 and margin=0.1 (nearly tied classes) is clearly different from an easy negative with S=10.0 and margin=0.9.

Evaluation for Phase 2

The margin metric m is a zero-cost addition — it is computed from the existing α tensor and should be added to profile_uncertainty immediately. It transforms the routing signal from one dimension (S) to two dimensions (S, margin), which substantially improves separation without any training changes.

C-EDL conflict calibration requires generating label-preserving transforms during training, which adds data pipeline complexity. Feasible in Phase 2 but better scheduled for Phase 3 robustness hardening.

Margin metric is free and should be added now. C-EDL transforms are Phase 3 work.

Issue 3 Brier = 0.250, Not Converged

PDF Diagnosis

Pages 1–2 and 7 address training stability directly:

"Use the standard EDL loss structure: data-fit + KL-to-uniform with annealing. This matters because otherwise you'll get either vacuous predictions everywhere or spiky overconfidence."

This is exactly the KL annealing fix already proposed. The PDF additionally mentions Re-EDL (page 2) as a simplification — removing parts of the classical loss to reduce tuning complexity and expose the prior weight as a single tunable hyperparameter.

PDF's Specific Addition

Meta-Policy Control (MPC, page 2): A policy network that dynamically adapts KL weight via bi-level optimization. Theoretically principled version of manual KL annealing, but significantly more complex to implement.

Evaluation for Phase 2

Re-EDL's simplification (dropping variance regularization, using prior weight as the single hyperparameter) is directly applicable. The current implementation uses both Brier loss and binary KL penalty — Re-EDL would drop one of these and replace both with a single prior-weight-scaled loss. This reduces the hyperparameter space from {KL_WEIGHT, S_REG_WEIGHT, OOD_LOSS_WEIGHT} to a single λ, which is easier to tune.

MPC is clearly out of scope — it requires a separate policy network trained via bi-level optimization.

Re-EDL simplification is worth considering as a replacement for Brier+KL. MPC is Phase 4+. Manual annealing is the correct Phase 2 action.

Issue 4 OOD Data is Word-Shuffled

PDF Diagnosis

This is the most extensively discussed issue. Pages 4 and 11 are explicit:

"Adversarial/far-OOD inputs can induce high Dirichlet strength S (low u), producing spurious confidence… this is the exact failure mode that would make your gating do the wrong thing."

The PDF recommends constructing OOD slices from: unseen verticals, new languages / code-mixed text, spam/SEO corpora, and adversarial lexical overlap negatives (linguistically similar but wrong).

Critically, the PDF also recommends adding a lightweight text-quality score as a separate feature: language ID confidence, character entropy, compression ratio, tokenization anomaly rate.

Evaluation for Phase 2

The OOD replacement (BeIR/SciFact, BeIR/FIQA) already identified is correct and aligns with "unseen verticals." The PDF adds two important extensions:

First, the spam/SEO corpus slice is specific to IR and is not covered by BeIR. Web spam and SEO-optimized pages with high lexical overlap to queries but no informational content are the most dangerous OOD failure mode for a retrieval system. These need to be in the OOD evaluation set before Phase 3 deployment.

Second, the text-quality score as an auxiliary feature is independent of EDL and cheap to implement:

def text_quality_score(text: str, tokenizer) -> float:
    """
    Cheap proxy for text coherence. Low score = gibberish/spam.
    Combines character entropy with tokenization anomaly rate.
    """
    from collections import Counter
    import math

    # Character entropy
    counts = Counter(text)
    total  = len(text)
    char_entropy = -sum(
        (c/total) * math.log2(c/total) for c in counts.values()
    )

    # Tokenization anomaly: ratio of [UNK] tokens
    tokens   = tokenizer.encode(text, add_special_tokens=False)
    unk_rate = tokens.count(tokenizer.unk_token_id) / max(len(tokens), 1)

    return char_entropy * (1 - unk_rate)  # high = coherent, low = gibberish

This score, combined with S and margin, gives a three-dimensional routing signal that is robust to the adversarial confidence failure mode.

BeIR replacement is confirmed correct. Spam/SEO slice is a Phase 3 requirement. Text-quality score is a cheap Phase 2 addition.

Issue 5 Theoretical Language

PDF Diagnosis

"Uncertainty shouldn't start from 'pick an entropy.' It should start from the decision and its loss."
"Treat Uepi and Ualea as features, not ground truth semantics."

The PDF does not recommend abandoning the epistemic/aleatoric language entirely — it recommends treating the decomposition as a feature panel rather than a ground-truth measurement.

Recommended Language Revision

Current LanguageEarlier DiagnosticPDF-Aligned
"epistemic uncertainty" "domain-boundary distance" Reducible uncertainty feature (Uepi)
"aleatoric ambiguity" (not addressed) Irreducible uncertainty feature (Ualea)
"vacuity" "free-energy proxy" Vacuity u = K/S — ignorance signal, not probability

The PDF's version is more precise because it preserves the useful distinction between vacuity (I have no evidence) and dissonance (I have contradictory evidence), which the simpler "domain-boundary distance" framing collapses. This distinction becomes critical in Phase 3 when routing needs to distinguish "I haven't seen this domain" from "this document is internally contradictory."

Issue 6 MC Dropout vs PDF Recommendation

PDF Diagnosis

MC Dropout is listed as a strong practical baseline (page 3) but with a specific caveat:

"Ensembles are great teachers (for distillation) but expensive at runtime. Bad fit for trillion-scale reranking unless you only apply them to a tiny tail."

The PDF's preferred path to epistemic signal is not MC Dropout at runtime — it is using ensembles offline to generate teacher uncertainty labels, then distilling those into the EDL head. This is cheaper at inference than MC Dropout while achieving similar calibration quality.

Evaluation for Phase 2

MC Dropout at inference (n=20 forward passes) costs 20× the compute of a single pass — for a Cross-Encoder ranker operating on large candidate sets, this is unacceptable at scale.

The PDF's offline distillation approach is strictly better: run an ensemble once on the training set (or a representative sample), extract uncertainty labels, add them as an auxiliary training target. At inference, the single EDL forward pass implicitly carries the distilled epistemic signal without additional compute.

However, for Phase 2 specifically — where the goal is validating the uncertainty gradient, not deploying at scale — MC Dropout is still useful as a diagnostic tool. Running MC Dropout offline on the test set to verify that the head shows higher variance on hard negatives than easy negatives provides empirical evidence that the EDL head is learning the right signal.

Use MC Dropout as offline diagnostic in Phase 2. Plan offline ensemble distillation for Phase 3 production inference.

New Finding: Dissonance / Conflict Metric

The dissonance/conflict metric (pages 8–9) is entirely new relative to the previous Phase 2 analysis. The PDF distinguishes:

Vacuity = low total evidence = "I haven't seen anything like this"

Dissonance = conflicting evidence across classes = "I've seen things like this but they point in opposite directions"

In the current results, hard negatives produce moderate S (5.60) but this single number cannot tell you why S is moderate — whether it is because the model has little evidence (vacuity) or conflicting evidence (dissonance). For a hard negative, dissonance is the correct interpretation: the model has evidence for both Relevant and Irrelevant. For OOD, vacuity is correct.

Computing Dissonance from α

def dirichlet_dissonance(alpha, eps=1e-12):
    """
    Dissonance: conflicting evidence across classes.
    High dissonance + moderate S → hard negative (aleatoric ambiguity).
    Low dissonance + low S      → OOD (vacuity / ignorance).
    From Subjective Logic formulation.
    """
    S = alpha.sum(dim=1, keepdim=True).clamp_min(eps)
    b = (alpha - 1.0) / S            # belief masses
    b = b.clamp_min(0.0)

    K    = alpha.shape[1]
    diss = torch.zeros(alpha.shape[0], device=alpha.device)
    for i in range(K):
        for j in range(K):
            if i != j:
                bal_ij = 1.0 - (b[:, i] - b[:, j]).abs() / (
                    b[:, i] + b[:, j] + eps
                )
                diss += b[:, i] * b[:, j] * bal_ij
    return diss

Adding dissonance to profile_uncertainty would immediately reveal whether the hard-negative S=5.60 is vacuity-driven or dissonance-driven — a distinction that is invisible in the current results and critical for validating that the model is learning the right uncertainty structure.
